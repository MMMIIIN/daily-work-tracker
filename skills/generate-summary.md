# Generate Summary Skill

작업 기록을 분석하여 프로젝트별 요약과 전체 요약을 생성합니다.

## 입력

작업 기록 마크다운 파일 내용 (예: `~/.claude/daily-work/YYYY-MM-DD.md`)

## 출력

### 프로젝트별 요약

각 프로젝트에서 수행한 주요 작업을 1-2문장으로 정리:

- 구체적인 기능명, 파일명, 작업 내용 포함
- 기술적 세부사항보다는 "무엇을 했는지"에 집중
- 작업 흐름이 보이도록 순서대로 나열

**예시**:
```
로그인 기능 구현, 세션 관리 추가 및 테스트 코드 작성
```

### 전체 요약

하루 전체 작업을 2-3문장으로 정리:

- 주요 프로젝트명 언급
- 핵심 작업 및 성과 요약
- 전체적인 작업 방향성 표현

**예시**:
```
shopping-app 사용자 인증 시스템 구현, admin-dashboard API 연동 작업 진행
```

## 요약 생성 가이드라인

### 프로젝트별 요약 작성 규칙

1. **작업 단위로 그룹화**: 비슷한 작업은 하나로 묶기
   - Bad: "로그인 버튼 추가, 로그인 API 연결, 로그인 에러 처리"
   - Good: "로그인 기능 구현 (버튼, API 연결, 에러 처리)"

2. **구체적인 명사 사용**: 추상적 표현 지양
   - Bad: "기능 개선"
   - Good: "다크모드 토글 기능 추가"

3. **동사로 시작**: 작업 내용을 동사로 시작
   - "구현", "추가", "수정", "개선", "제거", "리팩토링"

4. **길이 제한**: 최대 2문장, 100자 이내 권장

### 전체 요약 작성 규칙

1. **프로젝트 개수 고려**:
   - 1개: 해당 프로젝트 작업 상세히
   - 2-3개: 각 프로젝트 핵심 작업 언급
   - 4개 이상: 가장 중요한 2-3개 언급 + "등"

2. **성과 중심**: 과정보다 결과 강조
   - Bad: "여러 파일을 수정하고 테스트함"
   - Good: "사용자 인증 시스템 완성, 테스트 커버리지 80% 달성"

3. **연결성 표현**: 관련 작업은 연결하여 표현
   - "A 작업 후 B 진행" 또는 "A, B 연동 작업"

## 사용 예시

### 입력 (작업 기록)

```markdown
# 📅 2026-01-15 작업 기록

## 🔹 shopping-app
> `/home/user/projects/shopping-app`

- **[10:30]** 장바구니 기능 구현 요청
- **[10:45]** 상품 추가/삭제 로직 작성
- **[11:20]** 결제 페이지 연동
- **[11:45]** 버그 수정 및 테스트

## 🔹 admin-dashboard
> `/home/user/projects/admin-dashboard`

- **[14:00]** 사용자 목록 API 연결
- **[14:30]** 테이블 컴포넌트 개선
- **[15:00]** 페이지네이션 추가
```

### 출력 (요약)

**shopping-app 프로젝트 요약**:
```
장바구니 기능 구현 (상품 추가/삭제, 결제 페이지 연동), 버그 수정
```

**admin-dashboard 프로젝트 요약**:
```
사용자 목록 API 연동 및 테이블 UI 개선 (페이지네이션 추가)
```

**전체 요약**:
```
shopping-app 장바구니 및 결제 기능 완성, admin-dashboard 사용자 관리 화면 개선 작업 진행
```

## 호출 방법

이 스킬은 `/daily-sync` 명령어 실행 시 자동으로 사용됩니다.
Claude가 작업 기록을 읽은 후, 이 가이드라인에 따라 요약을 생성합니다.
